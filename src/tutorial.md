## ПЛАН

**Выбранная технология:** Go  
**Проект:** `git-contributions` — утилита для визуализации коммитов Git (ASCII?гистограмма)  
**Цель:** создание CLI-инструмента, модификация (творческий пункт), написание туториала, вклад в Open Source, видео?презентация, отчёт.

### Сводка этапов

| Этап | Часы | Основные задачи |
|------|------|-----------------|
| **1. Исследование и планирование** | 4–6 | Анализ аналогов, определение MVP (флаги `--repo`, `--days`), архитектура (UML Sequence). |
| **2. Разработка базовой версии** | 8–10 | Парсинг флагов (`flag`), вызов `git log`, группировка коммитов по дням, вывод ASCII?графика, обработка ошибок. |
| **3. Модификация проекта** | 4–6 | Добавление уникальной фичи (например, цветной вывод по количеству коммитов, топ авторов, экспорт в PNG). |
| **4. Техническое руководство** | 6–8 | Пошаговый туториал с кодом, UML?диаграммы (Sequence, Class), скриншоты, блок?схема алгоритма. |
| **5. Вклад в Open Source** | 16–20 | Найти проект (50+ звёзд), взять `good first issue`, исправить, создать Pull Request, задокументировать. |
| **6. Презентация и отчётность** | 8–10 | Видео (5–7 мин), финальный отчёт, публикация на GitHub Pages. |
| **Итого** | **~46–60** | |

### Ключевые требования к результату

- **Код:** `main.go` (или пакет) с флагами, вызовом Git, ASCII?графиком.
- **Творческая модификация:** одна дополнительная функция (цвет, топ авторов, PNG?экспорт, анимация).
- **Руководство:** Markdown?туториал с пошаговыми инструкциями, примерами кода, UML?диаграммами (Sequence, Class), скриншотами (4–6 шт.), блок?схемой.
- **Open Source:** Pull Request в сторонний проект, описание в `CONTRIBUTION.md`.
- **Видео:** демонстрация работоспособности, объяснение цели, задач, решения (5–7 минут).
- **Отчёт:** хронология этапов, индивидуальный план, выводы.

### Рекомендации

- Использовать только стандартную библиотеку Go (`flag`, `os/exec`, `time`, `strings`).
- Для диаграмм — Mermaid.js (можно встроить в Markdown).
- Видео записывать через OBS Studio, смонтировать в CapCut.
- Репозиторий опубликовать на GitHub Pages (`docs/index.html`).
