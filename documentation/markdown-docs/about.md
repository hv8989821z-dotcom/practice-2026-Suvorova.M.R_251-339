# Техническое задание

## 1. Цели системы

- Сокращение времени поиска научного руководителя с 2–3 недель до нескольких дней.
- Исключение неподходящих заявок за счёт автоматической фильтрации по кафедре/направлению.
- Обеспечение прозрачного взаимодействия студента и преподавателя после подтверждённой заявки.
- Формирование доверительного рейтинга преподавателей на основе оценок ВКР и отзывов студентов.

## 2. Роли пользователей и права доступа

| Функция | Студент | Преподаватель | Модератор |
|---------|---------|---------------|-----------|
| Регистрация по номеру телефона (SMS OTP) | Да | Да | Да |
| Загрузка подтверждающего документа | Да (зачётка) | Да (выписка о работе) | Нет |
| Проверка и подтверждение аккаунтов | Нет | Нет | Да |
| Настройка профиля и кафедр | частично | Да | Нет |
| Отправка заявок преподавателям | Да | Нет | Нет |
| Принятие/отклонение заявок | Нет | Да | Нет |
| Чат и вложения после match | Да | Да | просмотр по правам |
| Публикация отзывов | отправка на модерацию | Нет | модерация/публикация |
| Просмотр доказательств (фото оценки ВКР) | Нет | Нет | Да |
| Просмотр журналов действий | Нет | Нет | Да |

> Разграничение доступа реализовано по ролевой модели (RBAC). Проверка прав выполняется на сервере.

## 3. Основные бизнес-процессы

### 3.1 Регистрация и модерация аккаунта

1. Пользователь выбирает роль (студент или преподаватель) и регистрируется по номеру телефона.
2. Система отправляет одноразовый SMS-код (OTP) и подтверждает номер.
3. Пользователь заполняет профиль и прикрепляет подтверждающий документ.
4. Аккаунт получает статус «На проверке».
5. Модератор проверяет документы и принимает решение: «Подтверждён» или «Отклонён».
6. При отклонении указывается причина; пользователь может повторно отправить документы.

### 3.2 Поиск преподавателя и отправка заявки

- После подтверждения студент открывает список преподавателей.
- Система автоматически фильтрует преподавателей по кафедре/направлению студента.
- Студент видит только преподавателей, принимающих работы по его кафедре и доступных для новых запросов.
- Студент может отправить заявки нескольким преподавателям и отменить непринятую заявку.

### 3.3 Обработка заявки преподавателем

- Преподаватель получает push-уведомление о новой заявке.
- Преподаватель принимает или отклоняет заявку.
- Преподаватель может настроить лимит студентов и включать/выключать приём новых заявок.
- При принятии создаётся match и открывается чат «студент–преподаватель».

### 3.4 Завершение работы, оценка ВКР и отзыв

1. После завершения работы и выставления оценки ВКР студент может оставить оценку преподавателю и текстовый отзыв.
2. Студент прикрепляет фото/скриншот выставленной оценки ВКР (доступно только модератору).
3. Отзыв проходит модерацию и после одобрения публикуется в профиле преподавателя.
4. Оценка ВКР используется в расчёте рейтинга результатов ВКР преподавателя.

## 4. Функциональные требования (ключевые)

- **FR-01:** Регистрация и вход по номеру телефона с подтверждением через SMS OTP.
- **FR-02:** Поддержка ролей: студент, преподаватель, модератор.
- **FR-03:** Загрузка подтверждающего документа при регистрации.
- **FR-04:** Интерфейс модератора для проверки заявок на подтверждение аккаунта.
- **FR-05:** Хранение статуса аккаунта (на проверке / подтверждён / отклонён / деактивирован).
- **FR-06:** Студент может искать преподавателей и отправлять заявки.
- **FR-07:** Преподаватель может принимать/отклонять заявки, настраивать лимит студентов.
- **FR-08:** Создание чата после подтверждённого match.
- **FR-09:** Чат поддерживает обмен сообщениями и файлами (PDF, DOC, DOCX, ZIP до 30 МБ).
- **FR-10:** Push-уведомления по ключевым событиям.
- **FR-11:** Публикация отзывов студентов после модерации.
- **FR-12:** Расчёт и отображение двух рейтингов преподавателя.

## 5. Правила заявок, match и ограничения

### 5.1 Статусы заявок

| Сущность | Статусы |
|----------|---------|
| Аккаунт | Черновик; На проверке; Подтверждён; Отклонён; Деактивирован; К удалению; Удалён |
| Заявка студент → преподаватель | Отправлена; Отменена студентом; Принята; Отклонена |
| Отзыв | На модерации; Опубликован; Отклонён |

### 5.2 Ограничения по кафедрам и доступности

- В профиле преподавателя указывается список кафедр/направлений.
- В профиле студента указывается его кафедра/направление (подтверждённое).
- Студенту показываются только преподаватели, принимающие работы по его кафедре.
- Система запрещает отправку заявки, если кафедра студента не входит в список кафедр преподавателя (проверка на сервере).
- Учитывается настройка преподавателя: приём новых заявок включён/выключен, а также лимит студентов.

### 5.3 Правила match

- Match создаётся только при принятии заявки преподавателем.
- До принятия студент может отменить заявку.
- После match открывается чат с доступом для студента, преподавателя и модератора (по служебным правам).

## 6. Чат, файлы и уведомления

### 6.1 Чат

- История сообщений сохраняется.
- Удаление сообщений пользователями не предусмотрено.
- Сообщения доступны только участникам чата и модератору.
- Вложения отображаются в истории сообщений.

### 6.2 Вложения

| Параметр | Требование |
|----------|-------------|
| Допустимые форматы (MVP) | PDF, DOC, DOCX, ZIP (расширяемый список) |
| Максимальный размер | до 30 МБ на файл |
| Доступ к файлам | только участники чата и модератор |
| Подтверждение отзыва (фото оценки ВКР) | не публикуется, доступно только модератору |

### 6.3 Push-уведомления (минимальный набор)

- Новая заявка преподавателю.
- Заявка принята / отклонена.
- Новое сообщение в чате.
- Аккаунт подтверждён / отклонён модератором.
- Предупреждение о длительной неактивности и будущем удалении аккаунта.

## 7. Рейтинги преподавателей и отзывы

### 7.1 Два независимых рейтинга

| Рейтинг | Источник данных | Правило отображения |
|---------|----------------|----------------------|
| Рейтинг результатов ВКР | Итоговые оценки ВКР студентов преподавателя | Показывается всегда, даже при 1 оценке |
| Рейтинг преподавателя студентами | Оценки в опубликованных отзывах студентов | Показывается с количеством отзывов |

### 7.2 Правила расчёта

- Оценки ВКР учитываются по пятибалльной системе.
- Оценку/отзыв о преподавателе может оставить только студент, получивший итоговый результат за свой ВКР.
- В рейтинг результатов ВКР учитываются все работы (без ограничения по периоду).
- Рейтинг отображается всегда, например: «5.0 на основании 1 оценки».
- В рейтинг преподавателя студентами учитываются только отзывы, прошедшие модерацию.

### 7.3 Отзывы студентов

- Отзыв (текст + оценка) можно оставить после завершения работы и выставления оценки ВКР.
- Для отправки отзыва студент прикрепляет фото/скриншот оценки ВКР (доступно только модератору).
- Отзыв публикуется только после модерации.
- Если аккаунт студента удалён за неактивность, опубликованный отзыв сохраняется с ФИО студента и его группой.

## 8. Безопасность и обработка данных

- Аутентификация по номеру телефона с SMS OTP, ограничение частоты запросов.
- Серверное разграничение доступа по ролям (RBAC).
- Подтверждающие документы (зачётка, выписка) и фото оценок ВКР доступны только модераторам.
- Файлы чатов доступны только участникам и модератору.
- Журналируются: подтверждение/отклонение аккаунта, модерация отзывов, просмотр закрытых файлов (по возможности), критические изменения статусов.
- Хранение персональных данных (ФИО, группа, телефон, документы) с соблюдением применимых требований.

## 9. Неактивность и удаление аккаунтов

- Отслеживается неактивность пользователя (отсутствие входа/действий).
- Через 90 дней неактивности отправляется push-уведомление с предупреждением.
- Удаление аккаунта через длительный период неактивности (например, 1 год) настраивается в бизнес-логике.
- Перед удалением аккаунт переводится в статус «К удалению» / «Деактивирован» с повторным уведомлением.
- При удалении аккаунта студента опубликованные отзывы сохраняются.

## 10. Нефункциональные требования (MVP)

| Категория | Требование |
|-----------|-------------|
| Производительность | Типовые операции интерфейса без заметных задержек при штатной нагрузке. |
| Масштабируемость | Архитектура позволяет отдельно масштабировать API, хранение файлов и сервис уведомлений. |
| Надёжность | Сохранность истории сообщений и журналов; обработка ошибок загрузки файлов с понятным сообщением. |
| Безопасность | RBAC, серверные проверки, ограничение доступа к файлам, журналирование критичных действий. |
| Удобство использования | Понятные статусы, фильтрация по кафедре, явное отображение двух рейтингов. |
| Поддерживаемость | Логика статусов и бизнес-правила вынесены в явные сервисы/модули. |

## 11. Критерии приемки MVP (основные)

1. Регистрация по номеру телефона, подтверждение OTP, отправка аккаунта на модерацию с документом.
2. Модератор подтверждает/отклоняет аккаунт, действие фиксируется в журнале.
3. Студент видит только преподавателей своей кафедры и не может отправить заявку преподавателю другой кафедры.
4. Преподаватель может настроить кафедры, лимит студентов и доступность новых заявок.
5. Студент может отправить заявку нескольким преподавателям и отменить непринятую.
6. После принятия заявки создаётся чат с сохранением истории сообщений и вложений.
7. Работают push-уведомления по основным событиям.
8. Студент после оценки ВКР может отправить отзыв с подтверждающим фото; отзыв публикуется только после модерации.
9. В профиле преподавателя отображаются два рейтинга с количеством оснований (оценок/отзывов).
